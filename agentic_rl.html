Agentic RL

基于强化学习的智能体训练——将 LLM 作为可学习策略，嵌入感知-决策-执行循环，通过 RL 优化多步任务表现

LLM as Policy · 多步决策 · 奖励驱动

从 LLM 训练到 Agentic RL

核心问题：如何让智能体具备更强的推理能力?

传统 LLM 训练关注单步对话优化，而智能体需要多步序贯决策——Agentic RL 正是为此而生

🤖

Agentic RL 定义

将 LLM 作为可学习策略 $\pi_\theta(a|s)$，嵌入智能体的感知-决策-执行循环，通过强化学习优化多步任务表现。与传统的单步对话优化不同，Agentic RL 将智能体交互过程建模为马尔可夫决策过程（MDP），目标是最大化累积折扣奖励。

PBRFT vs Agentic RL：MDP 框架对比

维度 PBRFT（传统对齐） Agentic RL（智能体对齐）
决策模式 单步对话优化 多步序贯决策
状态空间 状态 = 固定 prompt 状态 = 动态演化的环境观测
行动空间 文本生成 文本 + 工具调用 + 环境操作
奖励信号 单步奖励 $r(s_0, y)$ 累积折扣奖励 $\sum_{t} \gamma^t r(s_t, a_t)$
优化目标 单轮回复质量 长期任务完成度

核心公式对比

PBRFT 目标
$$J_{\text{PBRFT}}(\theta) = \mathbb{E}_{s_0, y \sim \pi_\theta} [r(s_0, y)]$$
Agentic RL 目标
$$J_{\text{Agentic}}(\theta) = \mathbb{E}_{\tau \sim \pi_\theta} \left[\sum_{t=0}^{T} \gamma^t r(s_t, a_t)\right]$$

LLM 训练全景图

两阶段范式：LLM 的训练分为预训练和后训练两个阶段。预训练赋予模型语言能力，后训练让模型学会遵循指令、与人类对齐。

预训练：Causal Language Modeling

通过自回归方式学习语言的统计规律，预测下一个 token：

预训练损失
$$\mathcal{L}_{\text{pretrain}} = -\sum_{t=1}^{T} \log P(x_t | x_1, ..., x_{t-1}; \theta)$$

后训练三步骤

1

SFT 监督微调

教会模型遵循指令格式，学习基本推理能力

SFT 损失
$$\mathcal{L}_{\text{SFT}} = -\sum_{i=1}^{N} \log P(y_i | x_i; \theta)$$
→
2

奖励建模 (RM)

学习人类偏好，训练打分模型

Bradley-Terry 损失
$$\mathcal{L}_{\text{RM}} = -\mathbb{E}_{(x, y_w, y_l)} [\log \sigma(r_\phi(x, y_w) - r_\phi(x, y_l))]$$
→
3

RL 微调 (PPO)

用 RL 最大化奖励，同时约束不偏离参考策略

PPO 损失
$$\mathcal{L}_{\text{PPO}} = \mathbb{E}_{x, y \sim \pi_\theta} [r_\phi(x, y)] - \beta \cdot D_{KL}(\pi_\theta || \pi_{\text{ref}})$$

RLHF vs RLAIF

RLHF（人类反馈）

奖励模型从人类标注的偏好对 $(y_w, y_l)$ 中学习。优点：直接反映人类价值观。缺点：标注成本高、一致性难保证、扩展性受限。

RLAIF（AI 反馈）

奖励模型从AI（如 GPT-4）生成的偏好对中学习。优点：可扩展、成本低、一致性好。缺点：可能继承 AI 的偏见、信号质量受限于 AI 能力。

Agentic RL 六大核心能力

Agentic RL 使 LLM 从"被动生成器"进化为"主动决策者"，赋予六大核心能力：

R
推理 (Reasoning)

通过试错学习有效推理策略，发现训练数据中没有的推理路径。不再是简单模仿人类思维链，而是能够自主探索更优的解题路径。

关键：探索 vs 利用的平衡

T
工具使用 (Tool Use)

学会何时使用工具、选择哪个工具、如何组合多个工具。从"什么都自己算"进化为"善用外力"。

关键：工具选择的时机与组合策略

M
记忆 (Memory)

决定哪些信息值得记住、何时更新记忆、何时删除过时信息。有效记忆管理是长期任务成功的关键。

关键：记忆的存取与遗忘策略

P
规划 (Planning)

通过试错发现有效的行动序列，学会权衡短期和长期收益。不是简单的前向搜索，而是动态调整计划。

关键：短期收益 vs 长期回报的权衡

S
自我改进 (Self-Improvement)

识别自己的错误、分析失败原因、调整策略。这是从"被动接受反馈"到"主动自我优化"的质变。

关键：错误检测与策略修正能力

P
感知 (Perception)

理解多模态信息，提升视觉推理能力。从纯文本理解扩展到图像、表格、代码等多模态输入的处理。

关键：跨模态信息融合与理解

数据集与奖励函数

GSM8K 数据集

7,473 训练样本 + 1,319 测试样本 | 小学数学应用题，每题需要 2-8 步推理 | 广泛用于评估 LLM 的数学推理与多步决策能力

三种奖励函数

1️⃣

准确率奖励

最简单直接的奖励信号，判断最终答案是否正确：

$$r_{\text{acc}}(a, a^*) = \begin{cases} 1 & \text{if } a = a^* \\ 0 & \text{otherwise} \end{cases}$$

特点：简单直接但稀疏，模型难以获得有效梯度信号

2️⃣

长度惩罚

在准确率基础上惩罚过长回答，鼓励简洁：

$$r_{\text{length}} = r_{\text{acc}} - \alpha \cdot \max(0, l - l_{\text{target}})$$

特点：鼓励简洁推理，避免模型生成冗余步骤

3️⃣

步骤奖励

为正确的中间推理步骤给予奖励：

$$r_{\text{step}} = r_{\text{acc}} + \beta \cdot s$$

特点：鼓励可解释的推理过程，s 为正确步骤占比

组合策略

实际训练中，往往需要组合多种奖励信号来平衡多个优化目标：

策略 A 准确率 + 长度惩罚 —— 侧重正确且简洁的回答
策略 B 准确率 + 步骤奖励 —— 侧重正确且推理过程可解释
策略 C 三者平衡 —— 准确率 + 长度惩罚 + 步骤奖励，全面优化 
# 奖励函数组合示例
def combined_reward(answer, target, length, target_len, steps, correct_steps):
    # 准确率奖励
    r_acc = 1.0 if answer == target else 0.0
    # 长度惩罚
    alpha = 0.01
    r_length = r_acc - alpha * max(0, length - target_len)
    # 步骤奖励
    beta = 0.5
    step_ratio = correct_steps / max(steps, 1)
    r_step = r_acc + beta * step_ratio
    return r_length + r_step - r_acc  # 避免重复计算 r_acc

SFT 与 GRPO 训练

SFT 是基础

SFT（监督微调）教会模型任务格式和基本推理能力。没有 SFT 打底，RL 训练会非常不稳定——就像让一个连规则都不懂的新手去参加比赛。

LoRA：参数高效微调

通过低秩分解大幅减少可训练参数，使微调在消费级 GPU 上成为可能：

LoRA 低秩分解
$$\Delta W = BA, \quad \text{where } B \in \mathbb{R}^{d \times r}, A \in \mathbb{R}^{r \times k}, \quad r \ll \min(d, k)$$

GRPO vs PPO：核心对比

PPO（传统方案）

  • ❌ 需要 4 个模型：Policy + Reference + Value + Reward
  • ❌ 训练复杂且不稳定，4 个模型之间相互影响
  • ❌ 显存消耗巨大（Value Model 与 Policy 同等规模）
  • ❌ 超参数多，调参困难
  • ⚠️ Value Model 估计不准会影响训练效果

GRPO（精简方案）

  • ✅ 只需 2 个模型：Policy + Reference
  • ✅ 用组内相对奖励代替 Value Model
  • ✅ 显存消耗大幅降低
  • ✅ 训练更稳定，超参数更少
  • ✅ 同一问题生成多个回答，组内比较作为基线
GRPO 损失函数
$$\mathcal{L}_{\text{GRPO}}(\theta) = \mathbb{E}_{s,a \sim \pi_\theta} \left[ \frac{\pi_\theta(a|s)}{\pi_{\text{ref}}(a|s)} \cdot (r(s,a) - \bar{r}_{\text{group}}) \right] - \beta \cdot D_{KL}(\pi_\theta || \pi_{\text{ref}})$$

其中 $\bar{r}_{\text{group}}$ 是同一 prompt 生成的多个回答的平均奖励，作为基线替代 Value Model 的估计。这种方法既降低了计算成本，又避免了 Value Model 估计不准确的偏差。

# GRPO 训练核心代码示意
from trl import GRPOTrainer, GRPOConfig

# 配置 GRPO 训练参数
config = GRPOConfig(
    output_dir="./grpo_model",
    num_generations=8,          # 每个 prompt 生成 8 个回答
    per_device_train_batch_size=4,
    learning_rate=5e-7,
    beta=0.04,                 # KL 惩罚系数
    max_new_tokens=512,
)

# 使用自定义奖励函数
def reward_fn(completions, **kwargs):
    return [compute_reward(c, k) for c, k in zip(completions, kwargs["answer"])]

trainer = GRPOTrainer(
    model=sft_model,
    args=config,
    reward_funcs=reward_fn,

trainer.train()

模型评估与总结

评估指标三大类

📐 准确性指标

Accuracy 最终答案正确率，最直接的评估指标
Top-K 准确率 前 K 个采样中至少一个正确的比例
数值误差 预测值与真实值的数值偏差

⚡ 效率指标

平均长度 生成回答的平均 token 数
推理步骤数 完成推理所需的中间步骤数量
推理时间 从输入到输出所消耗的时间

🔍 质量指标

格式正确率 输出是否符合预期格式规范
推理连贯性 各推理步骤之间逻辑是否自洽
可解释性 推理过程是否清晰、可理解 
# 评估示例
def evaluate_model(model, eval_dataset):
    results = {"accuracy": [], "avg_length": [], "format_rate": []}
    for sample in eval_dataset:
        output = model.generate(sample["question"])
        # 准确率
        results["accuracy"].append(output == sample["answer"])
        # 平均长度
        results["avg_length"].append(len(output.split()))
        # 格式正确率
        results["format_rate"].append(check_format(output))

    return {k: sum(v) / len(v) for k, v in results.items()}

总结：为什么需要 Agentic RL?

核心逻辑链
→
因为预训练模型不会遵循指令 → 所以需要 SFT
→
因为SFT 只能模仿训练数据 → 所以需要 RL (GRPO)
→
因为PPO 太复杂 → 所以选择 GRPO
→
因为单步奖励不够 → 所以设计多步奖励函数
→
因为需要平衡多个目标 → 所以组合多种奖励函数

一句话总结：Agentic RL 将 LLM 从"单轮回复机器"升级为"多步决策智能体"，通过 SFT 打基础、GRPO 做强化、组合奖励做引导，让 AI 学会推理、规划、自我改进——真正具备解决复杂任务的能力。
