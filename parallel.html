四种并行策略：DP、MP、PP、TP

当单卡无法承载大模型时，如何切分模型与数据？深入理解四种分布式并行策略的核心原理与应用场景

核心问题：大模型（如GPT-4、Llama 3）参数量巨大，单张GPU显存无法容纳。如何让多张GPU协同工作？

两个切分维度：可以切分数据（不同GPU处理不同样本），也可以切分模型（不同GPU承载模型的不同部分）。

四种策略各有优劣，关键在于：切什么、怎么切、切完如何通信。

每张GPU上放置完整的模型副本，输入数据被切分到不同GPU上各自独立计算。

训练时：各卡独立前向+反向，然后同步梯度（AllReduce），更新所有副本。

推理时：各卡独立推理，无需通信，可并行处理多个请求。

💡 形象比喻：就像复印教材给学生。每个学生拿到完整的教材（模型），各自学习不同章节（数据）。学习结束后，大家交流笔记（梯度同步），统一进步。

⚠️ 串行执行：GPU1必须等GPU0完成

模型太大放不进单卡，把不同层放到不同GPU上。数据流串行经过各卡。

训练时：前向传播逐卡传递激活值，反向传播逐卡传递梯度。

推理时：请求串行经过各卡，延迟等于各卡处理时间之和。

💡 形象比喻：就像流水线工厂的每个工位。工人A做第一步，工人B做第二步...必须等上一步完成才能开始下一步。一个人慢了，所有人都在等。

✗ 缺点

在MP基础上，将数据分成多个micro-batch，像流水线一样让不同批次在不同Stage同时执行。

训练时：多个批次同时在不同Stage处理，减少GPU空闲。

⚠️ 通信密集：每层计算后需AllReduce/AllGather

把单层的权重矩阵切分到多张GPU上，如Attention的Q/K/V头分到不同卡。

训练时：每卡计算部分结果，然后通信合并。通信最密集。

推理时：各卡并行计算部分输出，最后拼接。适合高带宽互联（如NVLink）。

💡 形象比喻：就像多人合力抬一个大箱子。每人只抬箱子的一部分（切分权重），但必须同步用力（密集通信）。配合不好就会乱套。
✓ 优点
  • 突破单层显存限制
  • 单层可并行计算
  • 推理延迟较低
✗ 缺点
  • 通信最密集
  • 依赖高带宽互联
  • 通常仅限单机内

决策框架：什么时候用什么？

模型能放进单卡
→

DP 数据并行

最简单、最高效。训练时梯度同步，推理时完全独立。

模型放不进单卡
但能放进单机
→

TP + DP

TP切分大层（利用NVLink高带宽），DP提升吞吐。单机内TP通信效率高。

模型放不进单机
需要跨机部署
→

PP + TP + DP

TP处理单机内层切分，PP处理跨机层切分（通信少），DP叠加提升吞吐。这是大模型训练的标准组合。

推理场景
追求低延迟
→

DP 或 TP

模型能放进单卡用DP（延迟最低）；否则用TP（层内并行，延迟接近单卡）。避免PP（延迟累加）。

推理场景
追求高吞吐
→

DP + 批处理优化

多卡独立推理 + 动态批处理（continuous batching）。DP是推理吞吐的最佳选择。

核心要点总结

四种策略的本质区别

DP 切数据，不切模型

每卡完整模型副本。训练需梯度同步，推理完全独立。最简单高效，适合模型能放进单卡的场景。

MP 切模型层，串行执行

不同层放不同卡。数据串行流过各卡。效率最低，通常演化为PP。

PP 切模型层，流水执行

层分组+多batch流水。减少GPU空闲。适合跨机部署，与TP组合使用。

TP 切模型层内权重

单层权重矩阵切分。通信密集。依赖高带宽互联，适合单机内大层切分。

一句话记忆：DP切数据（简单）、MP切层串行（不推荐）、PP切层流水（跨机）、TP切权重（单机）。大模型训练标配 = PP + TP + DP。
